# Tanakh365 — Портал для ежедневного изучения Танаха

Tanakh365 — это современное веб-приложение, разработанное для того, чтобы помочь пользователям прочитать и изучить весь Танах (Еврейскую Библию) в течение одного года. Приложение интегрируется с Sefaria для получения текстов и комментариев, а также использует искусственный интеллект (Gemini) для создания тестов и улучшения переводов.

## Основные возможности

- **Ежедневный план чтения:** Четко структурированный график изучения Невиим и Кетувим на каждый день года.
- **Интеграция с Sefaria:** Доступ к оригинальным текстам на иврите и классическим комментариям (например, Штейнзальц).
- **Интерактивные тесты:** Проверка знаний после каждого урока с мгновенной обратной связью и объяснениями.
- **Система достижений:** Получение значков (badges) за завершение книг и сертификата по окончании годового цикла.
- **Двуязычный интерфейс:** Поддержка русского и английского языков.
- **Традиционная терминология:** Использование аутентичных еврейских имен и терминов (Моше, Йеошуа, Бней Исраэль и т.д.) в переводах.
- **Панель администратора:** Инструменты для кураторства контента, генерации тестов с помощью ИИ и управления ролями пользователей.

## Технологический стек

- **Frontend:** React 19, TypeScript, Vite.
- **Backend:** Express.js (Full-stack режим).
- **Стилизация:** Tailwind CSS 4, Lucide React (иконки).
- **Анимации:** Motion (ранее Framer Motion), canvas-confetti.
- **База данных и аутентификация:** Firebase (Firestore, Firebase Auth).
- **Искусственный интеллект:** Google Gemini API (@google/genai).
- **Работа с датами:** Hebcal (еврейский календарь), Luxon.

## Начало работы

### Предварительные требования

- Node.js (версия 18 или выше)
- Аккаунт Firebase
- Ключ API Google Gemini

### Установка

1. Клонируйте репозиторий.
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Настройте переменные окружения (см. раздел ниже).

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`.

## Переменные окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

- `GEMINI_API_KEY`: Ваш API ключ для доступа к моделям Gemini.
- `VITE_FIREBASE_API_KEY`: API ключ вашего проекта Firebase.
- `VITE_FIREBASE_AUTH_DOMAIN`: Auth домен Firebase.
- `VITE_FIREBASE_PROJECT_ID`: ID проекта Firebase.
- `VITE_FIREBASE_APP_ID`: App ID Firebase.
- `VITE_FIREBASE_FIRESTORE_DATABASE_ID`: (Опционально) ID базы данных Firestore.

## Структура проекта

- `src/components/`: Реакт-компоненты (Dashboard, Quiz, AdminView и др.).
- `src/services/`: Логика взаимодействия с внешними API (Sefaria, Gemini).
- `src/data/`: Статические данные, расписание чтения и контекст языка.
- `src/firebase.ts`: Конфигурация и инициализация Firebase.
- `server.ts`: Серверная часть на Express для обработки API и статики.
- `firestore.rules`: Правила безопасности для базы данных Firestore.
